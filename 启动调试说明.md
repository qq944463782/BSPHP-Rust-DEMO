# BSPHP Rust 演示 — 启动与调试说明

本目录为 **Cargo 工作区根目录**，包含：

| 目录 | 说明 |
|------|------|
| `bsphp.Rust.user` | 账号模式桌面演示（多 Tab、控制台） |
| `bsphp.Rust.car` | 卡模式桌面演示（卡密 / 机器码、主控制面板） |

---

## 一、环境要求

- **Rust 工具链**：建议 stable（[rustup](https://rustup.rs/) 安装）
- **操作系统**：macOS / Linux / Windows（图形界面依赖 `eframe`）
- **网络**：需能访问演示站 `demo.bsphp.com`（或你替换后的 AppEn 地址）

终端检查：

```bash
rustc --version
cargo --version
```

---

## 二、编译

在 **本目录**（含根目录 `Cargo.toml`）执行：

```bash
cd /Users/enzuzhou/Documents/BSPHP-Rust-DEMO
cargo build --workspace
```

仅编译某个包：

```bash
cargo build -p bsphp_rust_user
cargo build -p bsphp_rust_car
```

发布（优化）构建：

```bash
cargo build --workspace --release
```

---

## 三、运行（启动）

账号模式：

```bash
cargo run -p bsphp_rust_user
```

卡模式：

```bash
cargo run -p bsphp_rust_car
```

可执行文件位置（调试时可用）：

- `target/debug/bsphp_rust_user`
- `target/debug/bsphp_rust_car`

---

## 四、测试

```bash
cargo test --workspace
```

`bsphp.Rust.user` 与 `bsphp.Rust.car` 均包含编码相关单元测试；界面程序以手动联调为主。

---

## 五、调试

### 1. 命令行：崩溃栈

```bash
RUST_BACKTRACE=1 cargo run -p bsphp_rust_user
```

### 2. Cursor / VS Code：断点调试

1. 安装扩展 **CodeLLDB**（或 **rust-analyzer** 自带的调试支持）。
2. 先在本目录执行一次 `cargo build --workspace`，生成 `target/debug/` 下二进制。
3. 使用本仓库 **`.vscode/launch.json`** 中的配置：
   - **Debug bsphp_rust_user**
   - **Debug bsphp_rust_car**  
   在 `src/*.rs` 中打断点，按 F5 启动。

若扩展要求先编译，`preLaunchTask` 会执行对应 `cargo build`。

### 3. 仅构建再手动附加

```bash
cargo build -p bsphp_rust_user
lldb target/debug/bsphp_rust_user
```

（`lldb` 为 macOS/Xcode 命令行工具；Linux 可用 `gdb`。）

## 六、配置与对接后台

演示站参数写在各工程的 **`config.rs`**，修改后需重新 `cargo build` / `cargo run`：

- 账号模式：`bsphp.Rust.user/src/config.rs`（appid `8888888` 等）
- 卡模式：`bsphp.Rust.car/src/config.rs`（appid `66666666` 等）

须与 BSPHP 后台「当前应用」的 **AppEn 地址、mutualkey、RSA 密钥对** 一致；售卡/续费浏览器链接中的 **`daihao`** 也需与后台软件代号一致。

---

## 七、常见问题

| 现象 | 处理 |
|------|------|
| 初始化失败 / 连接失败 | 检查网络、防火墙、`config.rs` 中 URL 与密钥是否与后台一致 |
| 编译很慢 / 首次拉 crate | 正常现象，等待 `cargo` 完成；可配置国内 crates 镜像 |
| 界面无响应 | 网络请求在后台线程；若长时间 busy，查看是否阻塞在 TLS/代理 |
